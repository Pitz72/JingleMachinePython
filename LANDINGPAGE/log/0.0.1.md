# Log di Sviluppo v0.0.1 - Genesi del Sito

**Data:** 05 Dicembre 2025
**Autore:** Antigravity

## Obiettivo
Creare una landing page di "altissimo profilo estetico e qualitativo" per **Runtime Radio Live Machine Pro**, che rifletta la natura professionale e broadcast-grade del software.

## Stack Tecnologico
Per garantire prestazioni eccellenti e un'esperienza di sviluppo moderna, è stato selezionato il seguente stack:
*   **Core:** React 19 + TypeScript (tramite Vite).
*   **Styling:** Tailwind CSS v3.4.
*   **Build System:** Vite 7.

## Implementazione

### 1. Inizializzazione
Il progetto è stato inizializzato nella cartella `LANDINGPAGE` come applicazione standalone. Questo permette di mantenere il sito separato dal codice dell'applicazione Electron, facilitando il deployment e la manutenzione.

### 2. Design System "Premium"
Per rispondere alla richiesta di un tono "stiloso e professionale", sono state adottate le seguenti scelte di design:
*   **Palette Colori:**
    *   *Background:* `Zinc-950` (#09090b) - Un nero profondo e moderno, meno affaticante del nero assoluto.
    *   *Primary:* `Cyan-500` (#06b6d4) - Per richiamare l'identità tecnologica ed elettrica del software.
    *   *Text:* Bianco puro per i titoli, Zinc-400 per i testi secondari.
*   **Tipografia:** `Inter` (tramite system-ui) per la massima leggibilità e pulizia.
*   **Effetti Visivi:**
    *   Sfondi con gradienti radiali sottili per dare profondità.
    *   Effetto "Glassmorphism" (backdrop-blur) sulla barra di navigazione e sui pannelli.

### 3. Struttura della Pagina
La landing page è stata strutturata in sezioni logiche:
*   **Hero Section:** Titolo impattante con animazioni di ingresso (`fade-in`, `slide-up`) e CTA chiare ("Scarica per Windows").
*   **Interface Preview:** Un placeholder stilizzato per mostrare l'interfaccia dell'app.
*   **Features Grid:** Griglia a 3 colonne che evidenzia i punti di forza (Web Audio API, Latenza Zero, Cross-Platform).
*   **Tech Stack:** Una sezione dedicata alle tecnologie utilizzate (React, Electron, ecc.) per attrarre un pubblico tecnico.
*   **Footer:** Link essenziali e copyright.

## Note Tecniche e Risoluzione Problemi
Durante lo sviluppo è emerso un problema di compatibilità con **Tailwind CSS v4** (installato di default), che causava errori in fase di build con Vite.
*   **Soluzione:** È stato effettuato il downgrade a **Tailwind CSS v3.4.17** e configurato correttamente `postcss.config.js`. La build ora è stabile e priva di errori.
